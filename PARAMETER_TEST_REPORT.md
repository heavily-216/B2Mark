- **g값** (워터마킹 적용 비율의 역수): 3, 4, 5, 6, 7

### 총 테스트 조합
9 × 5 = **45가지 조합**

### 왜 짝수만 사용하는가?
```
짝수 k: Green Zone 개수 = Red Zone 개수 (정확한 50:50 분할) ✅
홀수 k: Green Zone ≠ Red Zone (불균형 분할) ❌

예시:
k=20 (짝수) → Green = 10개, Red = 10개 (50:50) ✅
k=21 (홀수) → Green = 10개, Red = 11개 (47.6:52.4) ❌
```

## 📎 부록: 기술 정보

### 신뢰도 점수 계산식
```
aux_score = (모든 비트의 Z검정 점수) / (비트 길이)

Z검정 점수 = (Green Zone 데이터 수 - 총 데이터 수/2) / √(총 데이터 수/4)
```

### Green/Red 분할 (짝수 k 기준)
```
k=20: 20개 구간 → Green 10개 + Red 10개 (50:50) ✅
k=22: 22개 구간 → Green 11개 + Red 11개 (50:50) ✅
```
